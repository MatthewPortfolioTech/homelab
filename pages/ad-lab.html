Active Directory Home Lab

Full setup of a Windows Server 2022 Domain Controller in VMware — covering AD DS installation, DNS, static IP, domain promotion, client join, and troubleshooting.

// contents

What's Covered

1. VMware Lab Setup
2. Windows Server Installation
3. Domain Controller Creation
4. Static IP Configuration
5. Active Directory Installation
6. Domain Promotion
7. Windows 11 Domain Join
8. DNS & Troubleshooting
9. Future Automation

1 VMware Lab Setup

Created the virtual environment using VMware Workstation. Set up a NAT network and an isolated lab environment before installing any server OS.

- Installed VMware Workstation
- Created NAT network
- Prepared isolated lab environment

2 Windows Server Installation

Installed Windows Server 2022 on the virtual machine. Mounted the ISO, completed the setup wizard, set an admin password, and installed VMware Tools for better performance.

- Mounted ISO
- Installed Windows Server 2022
- Set admin password
- Installed VMware Tools

3 Domain Controller Creation

Prepared the server to become a Domain Controller. Renamed the server to SYD-DC-01 and verified the system configuration before promoting.

- Renamed server to SYD-DC-01
- Verified system configuration

4 Static IP Configuration

Configured a static IP address so the server has a consistent address on the network — essential for a Domain Controller that other machines will rely on for DNS.

- Set IP manually
- Subnet: 255.255.255.0
- DNS: 127.0.0.1 (pointing to itself)

5 Active Directory Installation

Installed the AD DS (Active Directory Domain Services) role via Server Manager. DNS was installed automatically as a dependency.

- Added AD DS role via Server Manager
- DNS installed automatically as a dependency

6 Domain Promotion

Promoted the server to Domain Controller by creating a new Active Directory forest. After promotion, the server restarted and the domain was active.

- Created new forest
- Domain name: syd-lab.local
- Restart completed successfully

7 Windows 11 Domain Join

Joined a Windows 11 client machine to the syd-lab.local domain. Logged in with domain credentials to confirm the join was successful.

- Opened System Settings → Domain/Workgroup
- Joined domain: syd-lab.local
- Logged in with domain credentials successfully

8 DNS & Troubleshooting

Encountered DNS resolution issues during the domain join process. Fixed by correcting the DNS server settings on the client to point to the Domain Controller's IP instead of the router.

- Fixed nslookup failure on client
- Corrected DNS server to point at DC IP
- Verified domain resolution with nslookup syd-lab.local

9 Future Automation

Next phase is automating repetitive AD tasks using PowerShell and Group Policy Objects — removing the need to create users or apply settings manually.

- Bulk user creation scripts via PowerShell
- Automated AD provisioning workflows
- GPO deployment for security policies
