2
AD Onboarding & Offboarding Automation Portal
Completed

Built a full web-based onboarding and offboarding portal for Active Directory — replacing manual user management with a browser-based interface backed by PowerShell scripts and a Python Flask API running on the Domain Controller.

Environment: HomeLab.local · Domain Controller: SYD-DC-01 · April 2026

// what was built
  • Onboard-User.ps1 — creates a new AD user, places them in the correct OU based on department, and adds them to security groups
  • Offboard-User.ps1 — disables the account, resets the password to a random string, removes all group memberships, and logs the action
  • app.py — Python Flask API running on the DC, acts as the bridge between the web portal and the PowerShell scripts
  • ad-portal.html — web portal opened in any browser on the network by IT technicians, no installation required
  • portal-log.csv — auto-generated audit log recording every onboard and offboard action with timestamp and result
// request flow

IT tech opens portal in browser → fills in form → browser sends data to Flask API on DC → Flask calls PowerShell script → PowerShell creates or disables the user in AD → result is shown in the portal → action is logged to CSV.

1
Install Python & Flask on the Domain Controller

Downloaded Python 3 from python.org and installed it on the Domain Controller — making sure to tick Add Python to PATH before clicking Install Now. Then installed Flask via pip.

python --version          # Confirmed Python 3.14.4
pip install flask flask-cors
pip show flask            # Confirmed Flask 3.1.3
2
Create Scripts Folder & Files

Created the Scripts folder on the Domain Controller and saved all three files inside it.

New-Item -ItemType Directory -Path "C:\Scripts" -Force
# Created: app.py, Onboard-User.ps1, Offboard-User.ps1
Get-ChildItem C:\Scripts\   # Verified all files present
3
Test Flask & Open Firewall

Started Flask manually and confirmed it was running, then opened port 5000 in Windows Firewall so client PCs on the network could reach the API.

cd C:\Scripts
python app.py             # Flask confirmed listening on port 5000
# Tested http://localhost:5000/health → success

New-NetFirewallRule -DisplayName "AD Portal API" `
  -Direction Inbound -Protocol TCP `
  -LocalPort 5000 -Action Allow
!
Set Flask to Auto-Start on Boot

Created a Windows Scheduled Task to start Flask automatically on reboot.

Mistake First attempt used just python as the executable. After reboot the task failed with error code 2147942402 — the SYSTEM account couldn't find Python because it wasn't in the SYSTEM PATH.
Fix Ran where.exe python to get the full path. Deleted the broken task and recreated it using the full absolute path to python.exe. Flask auto-started correctly after the next reboot. 
where.exe python
# Result: C:\Users\Administrator\AppData\Local\Python\bin\python.exe

Unregister-ScheduledTask -TaskName "AD Portal Flask API" -Confirm:$false
# Recreated task with full python.exe path, running as SYSTEM at startup
Start-ScheduledTask -TaskName "AD Portal Flask API"

// code sample — Onboard-User.ps1

The PowerShell script that Flask calls when a new user is submitted through the portal. It builds the username, checks for duplicates, creates the AD account with all the correct properties, and adds them to the specified security groups.

POWERSHELL Onboard-User.ps1 
# Flask calls this automatically when the form is submitted

param(
    [Parameter(Mandatory)][string]$FirstName,
    [Parameter(Mandatory)][string]$LastName,
    [Parameter(Mandatory)][string]$Title,
    [Parameter(Mandatory)][string]$Department,
    [Parameter(Mandatory)][string]$Email,
    [string]$Office       = "Head Office",
    [string]$Manager      = "",
    [string]$Groups       = "",
    [string]$TempPassword = "Welcome@2024!"
)

# Build the username — first initial + last name → jsmith
$Username = ($FirstName.Substring(0,1) + $LastName).ToLower() -replace '[^a-z0-9]', ''

# If jsmith exists, try jsmith1, jsmith2, etc.
$Counter = 1
$Base = $Username
while (Get-ADUser -Filter "SamAccountName -eq '$Username'" -ErrorAction SilentlyContinue) {
    $Username = "$Base$Counter"
    $Counter++
}

# Create the AD user
New-ADUser `
    -SamAccountName        $Username `
    -UserPrincipalName     "$Username@HomeLab.local" `
    -Name                  "$FirstName $LastName" `
    -GivenName             $FirstName `
    -Surname               $LastName `
    -Title                 $Title `
    -Department            $Department `
    -EmailAddress          $Email `
    -AccountPassword       (ConvertTo-SecureString $TempPassword -AsPlainText -Force) `
    -Enabled               $true `
    -ChangePasswordAtLogon $true `
    -Path                  "OU=Users,OU=homelab,DC=HomeLab,DC=local"

# Add to security groups
foreach ($Group in ($Groups -split ",")) {
    Add-ADGroupMember -Identity $Group.Trim() -Members $Username
}

Write-Host "ONBOARDING COMPLETE: $Username created in HomeLab.local"
